## 7. 内置数据剥离范围

### 7.1 应进入官方内容包

- 物品基础定义及来源文本。
- 标签定义，以及物品声明的多值标签。
- 作物、种子、果树、鱼类和采集物定义。
- 普通怪物、BOSS 和有序怪物池配置；怪物定义与池成员关系使用独立注册表。
- 标准附魔定义、稀有度和标准效果参数。
- 命名掉落表及掉落条目。
- 烹饪、加工和其他标准配方。
- 商店定义、商品、价格、库存及显示分组。
- 与上述内容直接关联的图标、图片和音频资源引用。

### 7.2 应留在游戏框架

- 战斗回合、伤害、暴击、状态效果和击败结算算法。
- 掉落概率超过 100% 时的倍率计算。
- 矿洞和骷髅矿穴的楼层生成算法。
- 商店营业时间判断、购买上限和交易动作。
- 种植、收获、烹饪和加工的状态变更流程。
- 存档序列化、迁移、未知内容占位和恢复策略。
- Vue 页面、弹窗、路由和分页组件。

### 7.3 需要拆分的混合文件

迁移不以文件为单位，而以职责为单位。例如：

- `items.ts`：定义进入内容包，自动生成逻辑进入内容编译器，查找函数改为访问注册表。
- `monsters.ts` 与 `mine.ts`：前者作为纯怪物定义和旧池映射的叶子数据源，怪物与池投影进入内容包；后者保留旧导出适配器、楼层生成、缩放和随机算法。
- `weapons.ts`：附魔和武器定义进入内容包，附魔效果汇总和价格计算留在 domain。
- `shops.ts` 与 `ShopView.vue`：商店和商品定义进入内容包，营业判断和购买流程保留。

### 7.4 阶段 5 怪物迁移结果

- 21 个去重怪物定义进入 `taoyuan:monster`，掉落通过 `dropTableId` 引用 `taoyuan:drop_table`。
- 六个主矿洞区域池、六个 BOSS 楼层池和三个骷髅矿穴池进入 `taoyuan:monster_pool`。
- `MONSTERS`、`BOSS_MONSTERS`、`SKULL_CAVERN_MONSTERS` 和 `ZONE_MONSTERS` 继续由 `src/data/mine.ts` 原名重导出，作为兼容和回滚路径。
- `getFloor()`、弱化 BOSS、骷髅矿穴生成、矿洞 Store、矿洞页面和行会图鉴通过统一查询门面读取怪物池。
- 120 层区域映射、候选顺序、70% 弱化规则、深度阈值、25 层 BOSS 周期、缩放公式和随机调用位置仍属于框架且保持不变。

### 7.5 阶段 6 树木首切片结果

- 8 种果树和 3 种野树进入独立 `taoyuan:tree`，树苗、果实和采脂产物通过语义校验引用 `taoyuan:item`。
- `src/data/treeDefinitions.ts` 是旧格式树木数值的唯一静态叶子来源；`items.ts` 和官方适配器可以读取它，避免形成 `contentAccess -> staticAdapters -> fruitTrees/wildTrees -> contentAccess` 循环。
- `FRUIT_TREE_DEFS`、`WILD_TREE_DEFS` 继续由原模块原名重导出；`getFruitTreeDef()`、`getWildTreeDef()` 保留签名并在内部读取注册表兼容对象。
- 农场 Store、`useTreeActions.ts` 和 `ShopView.vue` 已通过统一门面读取树种、生长周期、季节产物、采脂周期和树苗信息。
- 种植、成长、换季、采脂器状态、收获、树木实例 ID 和存档字段仍属于框架且保持不变。
