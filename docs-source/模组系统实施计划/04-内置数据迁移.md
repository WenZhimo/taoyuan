## 7. 内置数据剥离范围

### 7.1 应进入官方内容包

- 物品基础定义及来源文本。
- 标签定义，以及物品声明的多值标签。
- 作物、种子、果树、鱼类和采集物定义。
- 普通怪物、BOSS 和有序怪物池配置；怪物定义与池成员关系使用独立注册表。
- 标准附魔定义、稀有度和标准效果参数。
- 命名掉落表及掉落条目。
- 烹饪、加工和其他标准配方。
- 商店定义、商品、价格、库存及显示分组。
- 博物馆分类、可捐赠条目、来源提示和里程碑奖励定义。
- 行会讨伐目标、捐献点数和等级经验阈值定义。
- 与上述内容直接关联的图标、图片和音频资源引用。

### 7.2 应留在游戏框架

- 战斗回合、伤害、暴击、状态效果和击败结算算法。
- 掉落概率超过 100% 时的倍率计算。
- 矿洞和骷髅矿穴的楼层生成算法。
- 商店营业时间判断、购买上限和交易动作。
- 行会贡献点余额、奖励领取状态、等级被动加成和旧存档迁移。
- 种植、收获、烹饪和加工的状态变更流程。
- 存档序列化、迁移、未知内容占位和恢复策略。
- Vue 页面、弹窗、路由和分页组件。

### 7.3 需要拆分的混合文件

迁移不以文件为单位，而以职责为单位。例如：

- `items.ts`：定义进入内容包，自动生成逻辑进入内容编译器，查找函数改为访问注册表。
- `monsters.ts` 与 `mine.ts`：前者作为纯怪物定义和旧池映射的叶子数据源，怪物与池投影进入内容包；后者保留旧导出适配器、楼层生成、缩放和随机算法。
- `weapons.ts`：附魔定义已拆到 `enchantmentDefinitions.ts` 并投影为 `taoyuan:enchantment`，武器定义已拆到 `weaponDefinitions.ts` 并投影为 `taoyuan:equipment` 的 `kind: "weapon"` 分支；附魔效果汇总、价格计算、随机附魔、掉落结算和装备实例仍留在 domain/store 兼容路径。
- `processing.ts`：加工机器制造定义已拆到 `processingMachineDefinitions.ts` 并投影为 `taoyuan:processing_machine`，加工配方定义已拆到 `processingRecipeDefinitions.ts` 并投影为 `taoyuan:processing_recipe`；机器运行队列、多任务酒坊/种子制造机、日更结算和存档槽位仍留在现有 Store/框架路径。
- `museum.ts`：博物馆分类、可捐赠条目和里程碑奖励已拆到 `museumDefinitions.ts` 并分别投影为 `taoyuan:museum_category`、`taoyuan:museum_item` 和 `taoyuan:museum_milestone`；捐赠扣料、奖励领取和存档字段仍留在 Store/框架路径。
- `guild.ts`：行会讨伐目标、捐献点数和等级经验阈值已拆到 `guildDefinitions.ts` 并分别投影为 `taoyuan:guild_goal`、`taoyuan:guild_donation` 和 `taoyuan:guild_level`；公会商店继续由 `taoyuan:shop_offer` 负责，贡献点状态、领取状态、被动加成和存档字段仍留在 Store/框架路径。
- `breeding.ts`：杂交品种定义已拆到 `breedingDefinitions.ts` 并投影为 `taoyuan:breeding_hybrid`；育种台数量、种子箱容量、加工天数、变异随机、属性波动、图鉴状态、种子实例和旧存档字段仍留在 Store/框架路径。
- `npcs.ts`：普通村民 NPC 定义已拆到 `npcDefinitions.ts` 并投影为 `taoyuan:npc`；好感事件定义已拆到 `heartEventDefinitions.ts` 并投影为 `taoyuan:heart_event`；普通任务模板已拆到 `questDefinitions.ts` 并投影为 `taoyuan:quest_template`；晨间随机事件定义已拆到 `farmEventDefinitions.ts` 并投影为 `taoyuan:morning_event`；普通任务实例生成/提交、好感状态和家庭/雇工算法仍留在 Store/框架路径或后续内容域。
- `hiddenNpcs.ts`：隐藏仙灵 NPC 定义已拆到 `hiddenNpcDefinitions.ts` 并投影为 `taoyuan:hidden_npc`；隐藏 NPC 好感事件已拆到 `hiddenNpcHeartEventDefinitions.ts` 并投影为 `taoyuan:heart_event`；亲密状态、供奉/求缘/结缘副作用和被动效果执行仍留在 Store/框架路径。
- `storyQuests.ts`：主线剧情任务定义已拆到 `storyQuestDefinitions.ts` 并投影为 `taoyuan:story_quest`；接取、进度判断、提交奖励、下一任务推进和旧存档字段仍留在 Quest Store/框架路径。
- `shops.ts` 与 `ShopView.vue`：商店和商品定义进入内容包，营业判断和购买流程保留。
- `market.ts`：市场行情分类定义已拆到 `marketDefinitions.ts` 并投影为 `taoyuan:market_category`；每日行情随机、供需插值、趋势分类、缓存和隔夜日志仍留在框架路径。
- `hanhai.ts`：通商积分兑换定义已拆到 `hanhaiDefinitions.ts` 并投影为 `taoyuan:hanhai_trade_exchange`；瀚海驿站固定/轮换商品已拆到 `hanhaiShopDefinitions.ts` 并复用 `taoyuan:shop_offer`；通商店铺升级定义已拆到 `hanhaiTradeShopDefinitions.ts` 并投影为 `taoyuan:hanhai_trade_shop_upgrade`；藏宝图奖励定义已拆到 `hanhaiTreasureDefinitions.ts` 并投影为 `taoyuan:hanhai_treasure_reward`；幸运轮盘结果与投注档位已拆到 `hanhaiRouletteDefinitions.ts` 并投影为 `taoyuan:hanhai_roulette`；通商积分余额、售货槽、兑换购买记录、特殊礼包/钱袋副作用、每周轮换洗牌、通商积分结算、藏宝图使用流程、轮盘转动动画和各类小游戏算法仍留在框架路径。
