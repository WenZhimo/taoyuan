# 平台加载、安全与信任

> [返回统筹说明](./README.md)

## 9. 平台加载策略

| 平台 | 官方内容 | 第三方内容来源 | 持久化方式 |
| --- | --- | --- | --- |
| Web | 随 Web 构建发布 | 文件选择器导入 | IndexedDB |
| Electron | 随应用发布的只读副本 | 程序目录 `mods/` 或导入，构建时同步备份恢复 | 程序目录 `userdata/` 中保存全局启用状态、信任库和缓存 |
| Android | 随 APK 发布 | 系统文件选择器导入 | 应用私有数据目录 |

数据包解析和校验必须是平台无关的纯 TypeScript。文件系统、文件选择器和 IndexedDB 只作为 `ContentPackageSource` 适配器存在。

Electron 应继续保持 `nodeIntegration: false` 和 `contextIsolation: true`。主进程不得把任意路径读取、命令执行或 Node API 暴露给数据包。

### 9.1 版本目录隔离

Electron 的每个游戏版本可以放在独立目录中运行，并拥有完全独立的程序、官方内容包、`mods/`、`userdata/`、全局模组配置、发布者信任库、存档、缓存和日志：

```text
桃源乡-2.4.0/
  taoyuan.exe
  mods/
  userdata/

桃源乡-3.0.0/
  taoyuan.exe
  mods/
  userdata/
```

“模组全局启用”只表示在当前程序目录/安装实例内全局启用，不表示整台电脑上的所有版本共享配置。新版目录导入旧版存档时必须先复制，不能直接读写旧目录中的原文件。发布说明应建议玩家保留旧版本目录，直到确认新版副本可正常游玩。

Web 和 Android 不能依靠可执行文件目录自然隔离。它们应为不同兼容环境使用独立存储命名空间或版本配置，并通过“复制后导入”进入新环境；不得让新版本直接覆盖尚未验证的旧环境存档。

### 9.2 资源类型与信任签名

资源是否允许加载由“文件类型策略 + 发布者信任 + 能力授权”共同决定。签名只证明发布者身份和文件完整性，不代表代码天然安全；即使来自受信任发布者，可执行插件仍必须经过沙箱、能力清单、超时和崩溃隔离。

首版资源策略：

| 资源级别 | 示例 | 默认策略 |
| --- | --- | --- |
| 被动数据与文本 | JSON、纯文本、Markdown | 未签名数据包可加载，但挂载器显示来源和完整性状态 |
| 被动图像 | PNG、JPEG、WebP、GIF | 校验真实 MIME、尺寸和解码上限后允许 |
| 音视频 | OGG、MP3、WAV、WebM、MP4 | 校验真实 MIME、时长、码率和解码上限后允许 |
| 字体 | WOFF2 | 限制文件大小、字体数量和作用域后允许，不得安装到系统 |
| 可主动执行或嵌入脚本的资源 | SVG、HTML、脚本化媒体、Shader | 数据插件默认拒绝；SVG 只有经过剥离脚本、外链和 `foreignObject` 的消毒/栅格化流程后才能作为被动图像 |
| 可执行代码 | JS、MJS、CJS、WASM、Node 扩展、DLL、EXE、BAT、PowerShell | 未受信任时一律拒绝；仅后续受信任事件/机制插件可按能力加载 |

包签名采用 Ed25519。构建数据包时生成规范化文件清单，记录每个文件的相对路径、大小和 SHA-256；发布者使用私钥对规范化清单签名。挂载器在解析内容前验证所有文件哈希、签名和清单完整性，签名不匹配、文件缺失或出现未列入清单的可执行文件时立即拒绝该包。

信任策略：

1. 官方发布密钥随游戏内置并默认信任。
2. 第三方签名包首次出现时，挂载器显示发布者公钥指纹、包能力和文件类型，由玩家显式信任该发布者；信任记录全局保存在 `userdata/` 或对应平台私有存储。
3. 未签名包只能包含允许的被动资源，挂载器必须持续显示“未验证发布者”状态。
4. 含可执行资源的包必须由已信任发布者签名，并由玩家逐项批准所需能力；未知、失效或被撤销的密钥不得执行代码。
5. 玩家可以撤销发布者信任。官方版本可以携带密钥撤销列表，但离线环境下仍以本地信任库和当前包签名为准。
6. 开发者模式可以允许本地调试包，但必须有独立开关、醒目标识和独立目录，不得把“开启开发者模式”静默视为永久信任发布者。

目录包还必须解析真实路径并拒绝通过 Windows junction、符号链接或其他重解析点越过包根目录。所有资源 URL 由挂载器生成，禁止数据包直接提供 `file:`、`javascript:` 或任意远程协议地址。
