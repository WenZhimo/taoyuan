# 资源限制默认值

> [返回统筹说明](./README.md)

## 原则

限制用于避免内存耗尽、主线程阻塞和压缩炸弹，不用于静默裁剪合法内容。超过限制时必须拒绝候选操作，报告包、文件、实际值和限制值，并保留原文件与旧配置。

这些是首版安全默认值。主程序可以通过受控开发配置降低或提高限制，但数据包自身不能申请放宽；公开发行版改变默认值时必须记录性能证据和兼容影响。

## 包与文件

| 项目 | 默认限制 |
| --- | ---: |
| 单个 ZIP 压缩大小 | 256 MiB |
| 单包解压后总大小 | 1 GiB |
| 单包文件数量 | 20,000 |
| 单文件大小 | 256 MiB |
| 压缩比 | 100:1 |
| 路径 UTF-8 长度 | 512 字节 |
| 目录深度 | 32 层 |
| 嵌套压缩包 | 禁止自动递归解压 |

解压前先检查中央目录和声明大小，解压过程中持续累计实际字节数。符号链接、junction 和其他重解析点一律拒绝。

## JSON 与 Schema

| 项目 | 默认限制 |
| --- | ---: |
| `manifest.json` | 1 MiB |
| 单个普通 JSON 数据文件 | 64 MiB |
| 单个配置 Schema | 2 MiB |
| 单语言包 | 16 MiB |
| `attributions.json` | 8 MiB |
| JSON 最大深度 | 64 |
| 单文件节点数 | 1,000,000 |
| 单数组条目数 | 250,000 |
| 单字符串 UTF-8 长度 | 1 MiB |
| 单包注册表条目总数 | 500,000 |

解析器必须在进入 Ajv 前执行大小和深度防护。官方压力测试使用至少 100,000 条定义，证明分阶段校验、进度反馈和取消行为有效。

### 怪物池权重展开

| 项目 | 默认限制 |
| --- | ---: |
| 单个怪物池原始条目数 | 250,000 |
| 单条 `weight` | 250,000 |
| 单个怪物池有效权重总和 | 250,000 |

三个数值统一复用 `MAX_JSON_ARRAY_ITEMS`，并由 TypeBox 结构边界、挂载前资源校验和运行时解析共同防守。刚好达到上限允许；超过上限产生 `PKG-LIMIT-001`，诊断必须携带 `poolId`、`actual`、`limit` 和 `fieldPath`。实现保留既有权重展开及条目顺序，不改变等权池随机结果或随机源调用次数；不得以静默截断、归一化或忽略权重代替拒绝。

## 图片、音频和视频

| 资源 | 默认限制 |
| --- | ---: |
| 单图片文件 | 32 MiB |
| 图片最大宽或高 | 16,384 像素 |
| 图片最大总像素 | 128 百万像素 |
| 单音频文件 | 128 MiB |
| 单音频时长 | 2 小时 |
| 单视频文件 | 256 MiB |
| 单视频时长 | 30 分钟 |
| 单包字体数量 | 16 |
| 单字体文件 | 16 MiB |

校验真实 MIME 和解码元数据，不只相信扩展名。媒体按需解码，启动挂载阶段不得一次性解码全部资源。

## 配置与插件私有数据

| 项目 | 默认限制 |
| --- | ---: |
| 单包安装级配置值 | 1 MiB |
| 单包存档级设置 | 1 MiB |
| 单插件 `payloadJson` | 16 MiB |
| 单存档全部 `pluginData` | 128 MiB |
| 单次私有数据净增长 | 8 MiB |
| 私有数据 JSON 深度 | 64 |
| 单插件私有数据节点数 | 1,000,000 |

容量按未压缩 UTF-8 字节计算。候选写入超限时保留旧信封；导入历史超限数据时保留原存档并拒绝可写加载，不得截断。

## 时间与响应性

| 项目 | 默认行为 |
| --- | --- |
| 单阶段无进度警告 | 5 秒后显示慢加载诊断 |
| 单包结构或语义校验软预算 | 30 秒 |
| 单包硬超时 | 120 秒 |
| 全部第三方包挂载硬超时 | 10 分钟 |
| 主线程连续工作预算 | 不超过 50 毫秒 |
| 进度更新间隔 | 100 至 250 毫秒 |

结构解析、Ajv 校验和引用扫描优先放入 Worker 或受控后台任务。达到软预算只警告；达到硬超时则取消候选挂载并报告当前阶段。官方内容包在发布前必须通过相同限制，不能依靠隐藏白名单绕过。

## 注册表缓存

| 项目 | 默认限制 |
| --- | ---: |
| 单个持久化快照 | 512 MiB |
| 当前安装实例缓存总量 | 2 GiB |
| 自动保留环境数量 | 当前环境加最近 2 个环境 |
| 临时缓存最长保留时间 | 24 小时 |

缓存超过总量时按最后成功使用时间清理未被当前 lockfile 引用的环境。最后成功环境和正在写入的事务目录不得被普通 LRU 清理。缓存超限或写入失败只影响加速能力，不能阻止完整挂载，也不能删除模组、配置或存档。缓存格式和恢复规则见 [注册表快照缓存](./15-注册表快照缓存.md)。

## 调整和兼容

- 降低限制可能使既有合法包失效，必须按兼容性变更处理。
- 提高限制必须先完成内存、磁盘和低性能设备压力测试。
- Web、Electron 和 Android 可以因平台能力采用更低的公开上限，但必须在导入前显示目标平台限制。
- 缓存、压缩和分页不能改变完整性校验所使用的原始计数。
