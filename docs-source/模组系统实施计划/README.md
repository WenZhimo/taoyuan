## 事实来源优先级

模组系统尚未发布时：

```text
现有代码和回归测试所证明的玩法行为
        → 本目录已经确认的架构决策
        → 阶段夹具和机器盘点
        → 示例与说明性文本
```

首个稳定 API 发布后：

```text
已发布稳定规范及规则编号
        → 从 TypeBox 生成的公开 JSON Schema
        → 官方一致性测试包及预期错误码
        → 参考实现
        → 示例和非规范性说明
```

若稳定规范、生成 Schema 和一致性测试互相矛盾，必须阻止发布并修复事实来源，不能由挂载器临时猜测。当前游戏内部 TypeScript 类型不是第三方公开契约。

## 已确认决策

- 官方内容和第三方数据包使用同一套注册表与校验流程。
- 注册表条目身份是 `(registryId, contentId)`，内容 ID 使用命名空间。
- TypeBox Schema 是官方公开内容契约的单一事实来源；Ajv 执行外部 JSON 校验。
- 同一安装实例中，同一 `PackageId` 只加载一个版本；版本冲突阻止启动。
- 模组在当前程序目录中全局启用，不按存档分别启停。
- 较高版本游戏可以暂存尝试旧模组，任一错误立即回滚相关依赖并停止游戏本体启动。
- 挂载器成功后才创建 `GameApp`；注册表在启动期构建，运行期冻结。
- 每次启动都经过挂载器；环境未变化时从已验证的持久化注册表快照恢复，缓存无效时自动完整挂载。
- Electron 的 `mods/` 和 `userdata/` 位于程序目录，重新构建必须保留二者。
- 原版旧存档只保证向原版新版本升级；模组存档跨环境不作保证。
- 插件私有状态进入 `pluginData`，通用设置进入 `packageSettings`，不得持续扩展根存档字段。
- 数据插件不执行未受信任代码；事件插件和机制插件属于后续阶段。

## 当前执行状态

阶段 0 和阶段 1 已完成首轮实现与自动化验收，详见 [执行状态](./执行状态.md)：

1. 已固定首批规范编号、错误码、ID 规则、哈希算法和生成式公开 Schema 流程。
2. 已将 `content-inventory.json` 扩展到 `src/data/` 导出符号级，并建立官方内容快照、哈希黄金向量和旧存档夹具。
3. 已实现 `ContentId`、`PackageId`、`RegistryTypeId`、通用 `Registry<T>` / `RegistrySet`、冻结生命周期、结构校验和语义校验。
4. 已建立首批 7 类注册表 Schema、最小合法/非法包夹具、旧静态数组兼容适配器和可序列化注册表快照 codec。
5. 尚未进入阶段 2 及之后的官方内容来源迁移、外载数据包加载器、模组管理界面或机制插件。

## 变更规则

- 架构决策修改必须同时更新受影响的主题文档、实施阶段、测试和风险。
- 公开字段变化必须先改 TypeBox Schema，再生成类型和公开 JSON Schema。
- 新的 MUST/MUST NOT 规则必须登记规则编号、错误码和一致性夹具。
- 更改默认资源限制必须记录原因、性能证据和兼容影响。
- 内容文件迁移后必须更新机器盘点状态，不能只修改说明文字。
- 重大变更同步更新 `../refactor-plan.md`；存档变化同步更新 `../save-compatibility.md`。
