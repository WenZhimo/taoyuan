## 5. 目标架构

```text
LauncherApp（挂载器）
读取全局模组配置 → 发现内容包 → 校验信任与完整性
        │
        ▼
注册表类型阶段
引擎内置类型 ─┐
授权事件/机制插件 ─┴→ 定义注册表类型 → 冻结类型目录
                                      │
                                      ▼
内容条目阶段
内容来源
├─ 内置官方内容包
├─ Electron 程序目录 mods/
├─ Web 导入并保存到 IndexedDB
└─ Android 文件选择器导入
        │
        ▼
ContentPackageSource
        │
        ▼
清单解析 → 依赖排序 → Schema 校验 → 注册 → 派生内容生成
        │
        ▼
跨表引用校验 → 冲突检测 → 冻结注册表条目
        │
        ├─ TagRegistry
        ├─ ItemRegistry
        ├─ MonsterRegistry
        ├─ EnchantmentRegistry
        ├─ DropTableRegistry
        ├─ RecipeRegistry
        └─ ShopOfferRegistry
        │
        ▼
生成不可变 RegistrySnapshot
        │
        ▼
GameApp（游戏本体）
Store / domain / composable / Vue 页面
```

挂载器与游戏本体应具有明确生命周期边界。挂载器负责模组管理、签名和权限确认、修复与校验、加载进度、错误日志、安全模式以及注册表快照构建；游戏本体只接收已经冻结且校验通过的 `RegistrySnapshot`，不得在运行过程中重新扫描或热卸载数据包。

模组启用状态是全局配置，不按存档分别启用。Web 将全局配置保存在 IndexedDB，Electron 保存在程序目录 `userdata/`，Android 保存在应用私有数据目录。存档只记录保存时实际使用的内容环境用于兼容性比较，不反向改变全局启用列表。

建议目录结构：

```text
src/

    launcherState.ts
    packageMountPipeline.ts
  engine/
    content/
      manifest.ts

      packageResolver.ts

      schemas/

        manifest.ts
        item.ts
        recipe.ts
      validation/

        semanticValidation.ts
      migrations/
      cache/
        cacheKey.ts
        snapshotCodec.ts
        snapshotCache.ts
    registry/
      Registry.ts
      RegistrySet.ts
      ids.ts
    saves/
      modMetadata.ts
      missingContent.ts
      pluginData.ts
  domain/
    combat/
    drops/
    farming/
    mining/
    shops/

content-packs/
  taoyuan-core/
    manifest.json
    settings.schema.json
    attributions.json

    items.json
    crops.json
    monsters.json
    enchantments.json

  valid/

    README.md

```

根目录 `schemas/` 保存从 TypeBox 生成并对外发布的 JSON Schema，属于只读构建产物，不得手工编辑。目录名称是建议，不应在一个提交中一次性建立所有空目录。只有对应阶段开始实现时才创建必要文件。

每次进程启动都经过挂载器，但不要求每次完整解析全部内容包。挂载器扫描并验证当前环境后，根据 `environmentHash` 尝试恢复持久化的编译注册表快照；命中时只重建运行时 Map、索引和只读包装，未命中或缓存无效时才执行完整挂载。具体契约见 [注册表快照缓存](./15-注册表快照缓存.md)。
