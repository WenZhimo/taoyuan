# 农田作物与树木系统

> 本文档基于当前 `src/` 源码重新调研生成，并已补充状态字段说明与更完整的核心机制流程。`docs/` 是构建输出目录，请继续在 `docs-source/game-systems/` 维护手写文档。

## 系统定位

说明地块、作物、浇水、肥料、季节、果树、野树、巨型作物和农场事件。

## 核心机制

- 农田的最小单位是 plot。地块状态记录是否荒地、已耕、已播种、可收获，以及作物 ID、成长天数、浇水、肥料、遗传种子等信息。玩家当天的浇水和种植只改变地块状态，真正成长在跨日结算，因此“操作反馈”和“生长结果”被刻意分开。
- `useFarmActions.ts` 负责把 UI 点击翻译成动作：根据当前工具、种子、地块和背包状态判断是耕地、浇水、播种、收获还是砍树，然后扣体力、推进时间、写入背包和技能经验。这样 `FarmView.vue` 只负责展示与交互，不需要直接处理所有经济结果。
- 跨日时 `useEndDay.ts` 调用农场每日更新。普通雨天会替代浇水，绿雨有额外成长效果，戒指和肥料能缩短成长或改变品质；乌鸦、害虫、巨型作物等事件也在这个阶段集中处理。由于这些规则互相叠加，新增成长加成时要明确叠加顺序。
- 树木分成果树和野树：果树偏长期产出，野树偏资源再生和砍伐。它们与普通作物共享农场空间感，但成长、产出和采伐规则独立；果树更像投资型产物，野树更像可再生资源点。
- 作物数据不是只服务农田本身。`crops.ts` 的种子、季节、售价、成长天数会被商店、市场、烹饪、加工、成就、图鉴和旅行商人共同引用，新增或改价时需要做全链路检查。

## 关键约束

- 当天动作只改变地块状态，作物成长和随机事件应留给跨日结算。
- 新增作物或树木时要同步种子、物品、商店、图鉴、成就和可能的加工/料理配方。

## 跨系统联动

- 与时间系统联动：动作消耗、跨日推进、每日重置或过晚边界需要保持一致。
- 与背包/经济联动：奖励、材料扣除、购买出售和容量失败需要走统一 store。
- 与长期进度联动：新增动作应考虑技能经验、成就统计、任务进度、图鉴发现或博物馆捐赠。

- 修改状态字段时，检查对应 `serialize` / `deserialize` 是否需要更新。
