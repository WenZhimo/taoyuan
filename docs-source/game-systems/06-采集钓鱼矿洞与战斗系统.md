# 采集钓鱼矿洞与战斗系统

> 本文档基于当前 `src/` 源码重新调研生成，并已补充状态字段说明与更完整的核心机制流程。`docs/` 是构建输出目录，请继续在 `docs-source/game-systems/` 维护手写文档。

## 系统定位

说明采集、钓鱼、蟹笼、矿洞楼层、矿石、怪物、战斗、公会和冒险奖励。

## 核心机制

- 采集、钓鱼、矿洞、战斗共享一套冒险成本模型：通常要消耗体力和时间，产出材料或战利品，并同步技能经验、成就统计、任务进度和图鉴发现。它们的共同风险是奖励很多，但背包容量和体力不足会让流程提前失败。
- 钓鱼由地点、季节、天气、时间、鱼竿、鱼饵和钓具共同决定可出现鱼池。开始钓鱼会抽取目标鱼和小游戏参数，完成后按表现发放鱼、宝箱、完美标记或蟹笼相关收获；完美钓鱼和钓具耐久也会进入后续判断。
- 矿洞是网格探索与回合战斗混合系统。揭开格子可能出现矿石、楼梯、陷阱或怪物；进入战斗后限制部分探索动作，战斗胜利才发放掉落、公会击杀和楼层推进资格。BOSS 或怪物未清理时，楼梯可见也不一定允许前进。
- 公会系统记录怪物遭遇、击杀、贡献点和目标领取。它依赖矿洞战斗结果，但又反向解锁奖励、商店或称号；因此战斗胜利不仅是矿洞内部事件，也是公会成长事件。
- 采集与矿洞还会产出烹饪、加工、建筑和博物馆材料。新增掉落物时要决定它是否可出售、可加工、可捐赠、可用于任务，以及是否计入发现图鉴。

## 关键约束

- 冒险动作必须同时考虑体力、时间、背包容量、技能经验、任务和成就。
- 战斗中要限制炸弹、移动和部分消耗品，避免绕过怪物或 BOSS 门槛。

## 跨系统联动

- 与时间系统联动：动作消耗、跨日推进、每日重置或过晚边界需要保持一致。
