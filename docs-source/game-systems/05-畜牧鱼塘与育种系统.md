# 畜牧鱼塘与育种系统

> 本文档基于当前 `src/` 源码重新调研生成，并已补充状态字段说明与更完整的核心机制流程。`docs/` 是构建输出目录，请继续在 `docs-source/game-systems/` 维护手写文档。

## 系统定位

说明鸡舍/畜棚、动物喂养、产出、疾病死亡、孵化、宠物、鱼塘和育种杂交。

## 核心机制

- 动物系统围绕建筑容量运转。先建造鸡舍/畜棚等建筑，再购买动物；每只动物保存年龄、好感、饥饿、生病、产出和名字，建筑等级决定容量、可养种类或自动化能力。建筑不是装饰，而是动物状态合法性的前置条件。
- 喂食是跨日结算前的关键前置条件。当天喂食、放牧、雇工/配偶帮忙会改变动物的过夜状态；未喂食可能导致生病、好感下降甚至死亡，已喂食则产出蛋、奶、毛等产品。这个时序要求喂食帮助必须发生在动物 `dailyUpdate()` 之前。
- 鱼塘是独立的长期养殖容器，鱼数、成熟度、密度、疾病和待领取产物共同决定每日结果。玩家投鱼、喂鱼、清理鱼塘和升级容量都会影响之后的产出；鱼塘不是背包里的鱼数量展示，而是独立状态机。
- 育种系统使用种子盒、工作站和谱系数据推进杂交发现。它与作物、鱼塘、成就和图鉴相连，重点不是立刻给奖励，而是记录新品种、遗传品质和发现进度。
- 畜牧、鱼塘和育种都属于“慢变量”系统：多数收益来自跨日积累。新增即时按钮时要小心，不能绕过年龄、成熟、喂食、疾病或杂交进度这些长期约束。

## 关键约束

- 动物喂食、雇工喂食和配偶帮助的时序必须早于动物每日更新。
- 育种和鱼塘是长期系统，UI 只发起动作，不应直接伪造成熟或产物。
